## 2. 활동별 핵심 팁

### 후기 공통 — 포인트 극대화 공식
```
사진 5장 이상 + 글자 400자 이상 = 무조건 8,000P (최고점)
동영상(15초 초과) 추가 = +2,000P
```
> ⚠️ 편집한 15초 영상은 인정 안 됨. 원본 동영상만 가능.

### 블로그 vs 인스타 비교
| 항목 | 블로그 | 인스타그램 |
|------|-------|----------|
| 포인트 | 5,000P | 2,000P |
| 최대 횟수 | 10회 | 15회 |
| 최대 합계 | **50,000P** | **30,000P** |
| 조건 | 글자 400자+ 사진 10장+ | 글자 300자+ 사진 3장+ |
| 효율 | ⭐⭐⭐ 높음 | ⭐⭐ 보통 |

→ **블로그 먼저 10회 소진 → 이후 인스타 15회 채우기** 순서 추천

## 6. 관리 툴 개발 계획

### 목표
포인트 활동을 매일 누락 없이 체크하고, 신청까지 이어지도록 도와주는 관리 툴

### Phase 1 — 웹 (현재)
- 기술: HTML/CSS/JS (정적 웹 페이지)
- 기능:
  - 한정 활동 체크리스트 (완료 표시 + 잔여 횟수)
  - 반복 활동 오늘의 체크리스트 (Daily/Weekly/Monthly)
  - 포인트 누적 현황 시각화
  - 포인트 신청 알림 (브라우저 알림)
  - 로컬 스토리지 기반 데이터 저장

### Phase 2 — 앱 확장 (추후)
- 기술: React Native 또는 Flutter (iOS + Android 동시 지원)
- 추가 기능:
  - 푸시 알림 (매일 아침/점심/저녁 웨딩수다 리마인더)
  - 댓글활동 시간 알림
  - 월말 포인트 신청 마감 알림
  - 활동 완료 시 진동/사운드 피드백
  - 포인트 적립 히스토리 캘린더
  - 남은 한정 활동 진행 현황 한눈에 보기

### Phase 1 → Phase 2 전환 고려사항
- 데이터 구조를 처음부터 JSON 기반으로 설계 → 앱 이식 용이
- 로컬스토리지 → 추후 Firebase 또는 Supabase 연동 고려
- 웹과 앱 간 데이터 동기화 가능하도록 API 구조 미리 설계

## 7. 주의사항

- 포인트 적립은 자동이 아님 → **카페 관리자 승인 후 처리** (요청일 기준 약 1개월 소요)
- 거절 시 **메모란 확인 후 재요청** 필요
- 모든 후기는 **제이웨딩에서 진행한 업체만** 인정
- 수원/종로 후기는 블로그/인스타 SNS 활동에 **포함 불가**
- 꽝 정보 포함된 자랑후기는 적립 불가
